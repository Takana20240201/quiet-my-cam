# quiet-my-cam / 静音カメラ

自分のためだけに作った、無音のカメラアプリ(PWA)です。

## なぜ作ったか

既存の無音カメラアプリには2つの不満がありました。

1. **データがどこに送られているか分からない。** カメラアプリは性質上、写真・位置情報・端末情報にアクセスできる。広告SDKやトラッキングが入ったアプリに、それを預けたくなかった。
2. **広告がストレス。** 撮りたい瞬間に全画面広告が出るような体験から離れたかった。

「信用できるアプリを探す」のではなく「信用が不要な構造のものを自分で作る」ことにしました。
このアプリはネットワーク通信を行うコードを1行も含みません。コードは全部ここに公開されているので、誰でもそれを確認できます。

## 特徴

- **無音撮影** — カメラの映像ストリームからフレームを直接取得する方式のため、シャッター音の仕組みを最初から通らない
- **通信ゼロ** — 外部リソースの読み込みなし、送信なし。画面上部に「通信 0」を常時表示
- **広告なし** — 当然
- **オフライン動作** — Service Workerにより、一度開けば圏外・機内モードでも起動する
- **アプリ内ギャラリー** — 撮影した写真はIndexedDBに永続保存。後からまとめて選んで写真アプリへ保存できる
- **前後カメラ切替・ピンチズーム**(対応端末ではセンサーズーム、非対応ならデジタルズーム)

## 使い方

1. このリポジトリのGitHub PagesのURLをスマホのブラウザで開く
2. カメラへのアクセスを許可する
3. 「ホーム画面に追加」すると、全画面のアプリとして起動できる

iOS Safariで動作確認していますが、Web標準APIのみで書かれているためAndroid ChromeやPCブラウザでも動きます。

## 仕組み

- 撮影: `getUserMedia` で映像ストリームを取得し、撮影ボタンで `<canvas>` にフレームを描画してJPEG化。通常の写真撮影APIを使わないため無音
- 保存: Web Share API経由でOSの共有シートを開き、写真アプリへ。非対応環境ではダウンロードにフォールバック
- 永続化: 撮影データはIndexedDBに保存。アプリを終了しても次回起動時に復元される
- オフライン: Service Workerがキャッシュ優先+バックグラウンド更新でファイルを配信

## 制約と注意

- 映像ストリームからの切り出しのため、標準カメラの写真モードより解像度は下がる(記録用途には十分)
- ナイトモードやポートレートなどOSの高度な撮影機能は使えない
- アプリ内ギャラリーは端末内の一時保管。OSがストレージ逼迫時にサイトデータを削除する可能性があるため、残したい写真は写真アプリへ保存すること(アプリ内にも注意書きあり)
- 無音での撮影は、使い方によっては法令(盗撮規制等)に触れます。自分の正当な用途にのみ使ってください

## 開発の背景

プログラミングはほぼ初めての状態から、**iPhoneだけ**で開発・公開しました。Macは一度も使っていません。

- 最初はSwiftでのネイティブアプリを検討したが、iPhoneだけでは開発環境が完結しないことが判明
- ブラウザのカメラAPIは仕組み上シャッター音が鳴らないと知り、Webアプリに方針転換
- GitHubのWeb UIからファイルをアップロードし、GitHub Pagesで公開
- 使いながら「連写した写真が保存できない」→ギャラリー機能、「圏外で使えない」→PWA化、「閉じると写真が消える」→IndexedDB、と不満駆動で改修

実装はAI(Claude)との対話で進めました。コードには日本語のコメントを付けてあり、何をしているか(そして通信していないこと)を読んで確認できます。
